PROJECT OVERVIEW
═══════════════════════════════════════════════════════════════════════════

The TestSense chatbot needed a backend system to validate test environments
before executing test scenarios. This implementation provides:

  1. Environment data collection (URL + credentials)
  2. URL reachability validation
  3. Login form detection
  4. Comprehensive error handling
  5. Integration with existing test pipeline

HOW THE CHATBOT USES THIS
═══════════════════════════════════════════════════════════════════════════

User Flow:
  1. User → "Test my login feature at https://myapp.com"
  2. Chatbot ↔ User: "Provide test credentials (valid + invalid)"
  3. Chatbot → Backend: POST /api/setup-environment
     {
       "environment": {
         "url": "https://myapp.com/login",
         "validEmail": "user@myapp.com",
         "validPassword": "SecurePass123!",
         "invalidEmail": "fake@test.com",
         "invalidPassword": "WrongPass456"
       }
     }
  4. Backend → Chatbot: {"reachable": true, ...}
     ("✓ Environment ready!")
  5. Chatbot → Backend: POST /api/generate-tests
     ("Generating test scenarios...")
  6. Chatbot → Backend: POST /api/run-tests
     (using validated environment)
  7. Chatbot → User: "5 tests passed, 1 failed (83% pass rate)"
